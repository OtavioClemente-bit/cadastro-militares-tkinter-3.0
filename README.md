# SIGFUR Tkinter

Versão legada do **SIGFUR**, desenvolvida em **Python** com **Tkinter**, criada para automatizar rotinas administrativas, organizar dados locais e reduzir trabalho manual em processos internos.

Este projeto representa a primeira fase de evolução do SIGFUR: uma aplicação desktop funcional construída com Python, interface gráfica, persistência local e módulos especializados para demandas administrativas reais.

> **Atenção:** este repositório deve conter somente código limpo e dados fictícios. Bancos reais, documentos pessoais, contracheques, boletins, planilhas sensíveis e arquivos internos não devem ser publicados.

---

## Objetivo

Criar uma ferramenta desktop para centralizar rotinas administrativas que antes dependiam de planilhas, arquivos soltos, conferências manuais e retrabalho.

A versão Tkinter serviu como base para validar fluxos, entender necessidades reais dos usuários e evoluir a arquitetura que posteriormente foi migrada para uma versão mais robusta em **C# WPF**.

---

## Principais módulos

- Cadastro e listagem de militares

- Boletim do furriel

- Pensão judicial
- Lembretes e tarefas administrativas
- Ferramentas de PDF
- Geração de documentos
- Conferências e relatórios internos

## Evolução do projeto

A versão Tkinter foi essencial para validar o conceito do SIGFUR e entender quais funcionalidades realmente geravam valor no dia a dia.

Com o crescimento do sistema, o projeto passou por uma evolução natural para uma versão em **C# WPF**, com foco em melhor arquitetura, interface mais profissional, organização por serviços, repositórios, modelos e maior capacidade de manutenção.

## Segurança e privacidade

Por se tratar de um sistema criado para apoiar rotinas administrativas reais, alguns arquivos do projeto original podem conter informações que não devem ser publicadas.

Antes de qualquer atualização de código neste repositório, devem ser removidos:

- Bancos de dados reais
- Arquivos `.db`, `.sqlite` e backups locais
- Documentos pessoais
- Planilhas sensíveis
- Contracheques
- Boletins ou PDFs internos
- Pastas `__pycache__`
- Ambientes virtuais
- Arquivos temporários
